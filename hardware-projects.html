Mortal Mayhem

An emulation of the retro classic Arcade Game Mortal Kombat

This 2-player handheld game is powered by the TI TIVA TM4C123G lauchpad. Custom-built hardware includes buttons for menu item selection/ attack, DACs for sound output, ADCs for joystick sampling, wireless receiver and transmitter antennas for serialized player-to-player communication, and an LCD screen for display. The main game routine is programmed in C. C++ modules are used to initialize hardware and communication protocols. Hardware interrupts are triggered on player input and are serviced using interrupt service routines. Busy-Wait threads and sempahores are used for synchronization of interrupts, enabling concurrency. Bounding Volume Hierarchy(BVH) algorithm is used for collision detection to process events where a player strikes the other and adjust health bar. This algorithm encloses the player sprites into geometric shapes and assesses overlap. Race conditions are caused by 2 players striking at the same time, which are solved by a randomization algorithm which arbitrarily sets priority. SPI communication module is implemented for communication with LCD screen. Serialization is configured using queues for firt-in-first-out data retrieval. Error codes are used error-checking and debugging. Images are stored as hexadecimal pixel matrices, and are loaded into buffers for display. Sounds/ music are stored as WAV files comprising discretely sampled pulses and are read periodically for playback.

Printed Circuit Board

Board powering a DC motor from a solar panel, competing with other boards to deliver the maximum power to the motor. Voltage dividers and current sensing resistors measure the solar panel's voltage (V_solar) and current (I_solar), with operational amplifiers conditioning these signals. The microcontroller reads these measurements with custom ADC and runs the Perturb & Observe (P&O) algorithm to determine the optimal operating point. The microcontroller also dynamically configures a programmable gain amplifier (PGA) to adapt to varying signal levels, ensuring accurate measurements. The control signal is output to a DAC, which converts it to an analog voltage for a voltage-controlled PWM generator. This PWM signal controls two MOSFETs in a half-bridge configuration, with an additional analog feedback loop using a precision voltage reference and a low-pass filter to stabilize and fine-tune the PWM signal for more efficient power delivery. The PCB is designed and fabricated using KiCAD, ensuring all components are correctly placed and routed. Testing involves verifying signal measurements, microcontroller functionality, PWM output, and overall system performance, including the adaptive PGA configuration and the analog feedback loop's effectiveness, under varying solar conditions to ensure maximum power delivery and enhanced system stability.
